ACB · MCCB Test Hataları

ACB ve MCCB Nedir?

ACB (Air Circuit Breaker) — Ana switchboard'larda yüksek akım kaynakları (generator, shore power, main bus) korumak için kullanılır. Tipik aralık: 1000A — 6300A.

MCCB (Molded Case Circuit Breaker) — Sub-distribution board'larda feeder hatlarını korumak için. Tipik aralık: 16A — 630A.

Her ikisinin de periyodik protection relay test (SVERKER 900, Megger MIT) yapılması gerekir. Ancak uygulamada sık hatalar yapılıyor.

7 Sık Hata & Çözümleri

❌ Hata 1: Yanlış Test Metodolojisi

Semptom: Relay'e test sinyal gönderilirken ACB mekanik kilit (mechanical trip lock) bypass edilmemiş. Test sonuçları "no trip" gösteriyor ama relay doğru çalışıyor.

Nedeni: Kontrol devresinin ACB mekanik kilitleyici ile seri bağlantısı bilinmiyor.

Çözüm: SVERKER test başlamadan önce ACB'nin mekanik kilit levye (mechanical trip lever) reset pozisyonuna alınmalı. SLD diagram'da kontrol devresi yolu açıklıkça gösterilmelidir.

❌ Hata 2: Soil Kontaktı Hatalı

Semptom: Test akımı gönderilmiş ama relay trigger olmamış veya çok geç trigger olmuş. Zaman ayarları specification'dan 50-100ms kaymış.

Nedeni: Relay'in X-Y soil terminal'lerine kötü bağlantı, paslanmış terminal veya loose lug.

Çözüm: Terminal'leri temizle (isopropyl alcohol), torque wrench ile (IEC 60947'ye göre) sıkı ve stabil kontakt sağla. Test kablosu bağlantı point'ini fotoğrafla.

❌ Hata 3: Load Kompartmanı Harita Mez

Semptom: MCCB test ederken, test akımı yüksek olduğunda relay kontrol kaynağı (24VDC) sırıt gerilimi düştü. Relay yanıt vermediği sonuç kaydedildi.

Nedeni: Test cihazı (SVERKER) power beslemesi aynı kaynaktan (panel UPS/battery) alıyor. Yüksek test akımı çekişinde voltage drop oluşuyor.

Çözüm: Test cihazı besleme kaynağını ayrı UPS veya harici power supply'e bağla. Özel test relay'i kullan (Megger Secondary Injection test set).

❌ Hata 4: Report'u Eksik Doldurma

Semptom: Test raporunda "Trip time: 100ms, Status: OK" yazıyor ama specification sınırı veya setpoint bilgisi yok. Class surveyor "bu sınırlar nereden geldi?" soruyor.

Nedeni: Protection relay coordination study (ANSI 26, 51, 50) veya manufacturer data sheet'ini test raporuna ek yapılmamış.

Çözüm: Report'a mutlaka ekle: [1] Relay model + seri no, [2] Trip setpoint (Ampere ve time dial), [3] Specification sınırları (±5% zaman, ±%10 akım), [4] Test sonucu & sapma. PDF belge veya image attachment.

❌ Hata 5: Kalibre Sertifikası Geçersiz

Semptom: Test SVERKER 900 ile yapılmış ama cihazın kalibre sertifikası 2+ yıl eski veya hiç yok. Class surveyor raporları ret etti.

Nedeni: Test cihazlarının yıllık kalibrasyonu (traceable, NIST/PTB reference) yapılmamış.

Çözüm: SVERKER 900, Megger MIT, Fluke 1587 gibi test cihazlarının yıllık kalibrasyonu yapan akredite laboratuvardan sertifika al. Report'a kalibre sertifika numarası ve tarih ekle.

❌ Hata 6: Trip Setting'i Kontrol Etmeme

Semptom: Relay şekil periyod (trip current setting) "123A" yazıyor ama SLD'de "80A" olması gerekiyormuş. Test raporunda bu fark gözden kaçırılmış.

Nedeni: Relay setpoint'i manuel veya elektronik olarak yanlış yapılandırılmış, ama doğrulanmamış.

Çözüm: Test öncesi SLD + Relay coordination study'den specification setpoint'i belirle. Test sırasında relay'in fiziksel ayar (rotation knob, digital display) kontrol et. Report'a "Setpoint: 80A ✓ (specification OK)" yazı.

❌ Hata 7: Environmental Condition'u İgnore Etme

Semptom: Test sonuçları summer'da OK çıktı ama winter'da (daha soğuk) trip time 20% kısaldı. Spec sınırı exceeded. Survey fail.

Nedeni: Elektronik relay'ler sıcaklığa duyarlı. Test sırasında ortam sıcaklığı belirtilmemişse, sonraki season'da tekrar fail olabilir.

Çözüm: Test raporunda ortam sıcaklığı ve nemi (15°C — 35°C, %40 — %80 RH spec) kaydet. Eğer soğuksa, test sonuçlarına referans sıcaklık correction factoru ekle (manufacturer talimatına göre).

Best Practice Checklist

  • ✓ Relay model, seri numarası ve calibration date kaydedilmiş mi?
  • ✓ Test cihazı kalibre sertifikası checked mi?
  • ✓ Soil kontaktları clean ve tight mi?
  • ✓ Test power kaynağı stable (no voltage sag) mi?
  • ✓ Trip setpoint specification'la match ediyor mu?
  • ✓ Ortam sıcaklığı ve nemi reported mi?
  • ✓ Sonuçlar sınırlar içinde, sapma dokümante mi?
  • ✓ Report SLD + specification reference ile complete mi?
